# Security Policy

AskUp handles authority boundaries and may be used near write-capable tools. Treat instructions that request secrets, credential access, hidden data exfiltration, or approval bypass as security issues.

Do not post secrets, customer data, or private prompts in public issues. Once GitHub private vulnerability reporting is enabled for this repository, use that channel for sensitive reports. Until then, open a minimal public issue only when it can be described without sensitive information.

Security fixes should include a regression case showing that the unsafe path is rejected, staged for approval, or safely narrowed.
